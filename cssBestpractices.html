20 CSS Best Practices

  1. 1. Create HTML first using a semantic wire frame and meaningful names for ID in class attributes.
  2. 2. Learn the language of CSS: selector, declaration, and property.
  3. 3. Learn the CSS syntax: the characters required to write accurate, readable styles.
  4. 4. Put global styles in the external style sheet. Use embedded and in-line styles only one appropriate.
  5. 5. Right styles in a sensible order starting with reset code.
  6. 6. Group styles that apply to similar Contin such as headings, lists, or forms together.
  7. 7. Put each declaration on it online to make the code easy to read.
  8. 8. Save your work and refresh your webpage often to observe incremental updates.
  9. 9. Use comments to separate sections of your style sheet.
  10. 10. Use comments to document the author, latest update, or anything else that is important.
  11. 11. Use combination selections such as h1, h2, h3 to code each declaration only once.
  12. 12. Use combination properties such as margin when appropriate to cut down on the number of declarations.
  13. 13. Study and truly understand the more difficult concepts of CSS such as flow and blocks versus in-line elements.
  14. 14. Use an outer wrapper and the margin: 0 auto; declaration to enter content within the outer wrapper.
  15. 15. Measure the width and % to make the containers responsive.
  16. 16. Measure tax and ems or % to make the text responsive.
  17. 17. Use firebug or similar web development tool to analyze and debug errors.
  18. 18. When debugging, use comments to “common out“ a rule that you are modifying (instead of deleting it).
  19. 19. Validate all HTML and CSS.
  20. 20. Never stop learning.
